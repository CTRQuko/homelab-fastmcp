# Code of Conduct

This project adopts the **Contributor Covenant, version 2.1**, without
modification. The full text is published at:

<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

A short summary in our own words:

- Treat other contributors as colleagues. Disagreement is fine;
  contempt isn't.
- Reviews focus on the code, the design, the issue — not on the
  person.
- New contributors are welcome. *"Why didn't you read the docs?"* is
  not an answer.
- The maintainer makes the final call on scope. Push-back is fine,
  pile-ons aren't.

## Reporting

For Code of Conduct issues, contact the maintainer privately. GitHub's
private vulnerability reporting feature works as a private channel
even for non-security reports — open the repo, click *Security* →
*Private vulnerability reporting*.

The maintainer responds within a week. Confidentiality is respected
unless the reporter says otherwise.

## Enforcement

Standard Contributor Covenant ladder applies (correction → warning →
temporary ban → permanent ban). The exact wording lives in the
upstream document linked above.
